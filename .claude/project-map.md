# Project Map: create-claude-repo-memory

Generated by: create-claude-repo-memory

## Purpose

An npm CLI package (`create-claude-repo-memory`) that scaffolds Git-tracked Claude project memory files into any repository. Running `npx create-claude-repo-memory` generates `CLAUDE.md` and a `.claude/` directory of structured Markdown templates that Claude reads at the start of every session. Published to the public npm registry at v0.1.0.

## Tech Stack

- **Runtime:** Node.js ≥18
- **Module system:** ESM (`"type": "module"` in package.json)
- **CLI framework:** commander ^12.1.0 (only dependency)
- **Language:** JavaScript — no TypeScript, no transpilation, no build step
- **Package manager:** npm
- **Registry:** npm public registry

## Entry Points

- **CLI:** `bin/cli.js` — invoked as `npx create-claude-repo-memory` or `create-claude-repo-memory [init]`
- **Programmatic (internal):** `src/init.js` exports `initClaudeRepoMemory({ rootDir, force, updateGitignore })`

## Major Domains

### CLI (`bin/cli.js`)
Parses `--force`, `--no-gitignore`, `--cwd`. Both the root action and the `init` subcommand call the same `runInit()`. Options are attached to each command via the shared `addInitOptions()` helper. Prints the created/overwritten/skipped summary and next-steps message.

### Init orchestration (`src/init.js`)
Owns `FILE_LIST` — the authoritative mapping of `templates/generic/` source files to target paths. For each entry: reads template, applies variables, checks existence, creates parent dirs, writes file. Calls `updateGitignoreFile` if enabled. Returns `{ created, overwritten, skipped, gitignoreStatus }`.

### Template resolution (`src/templateResolver.js`)
Replaces `{{PROJECT_NAME}}` and `{{GENERATED_BY}}` via `replaceAll`. Throws on any remaining `{{...}}` pattern after substitution — guards against template drift.

### Gitignore management (`src/gitignore.js`)
Appends a fixed block to `.gitignore`. Idempotency is checked by looking for `.claude-local/` in the existing file. Creates `.gitignore` if absent.

### Templates (`templates/generic/`)
Language-agnostic Markdown. All sections default to `Needs verification`. Two variables: `{{PROJECT_NAME}}` and `{{GENERATED_BY}}`. These are what end users receive.

## Critical Files

- `bin/cli.js` — must have the shebang; npm sets execute permission on publish
- `src/init.js` — `FILE_LIST` controls exactly what gets generated; changing it is a breaking change
- `src/gitignore.js` — `BLOCK_MARKER` (`.claude-local/`) is the idempotency key
- `templates/generic/CLAUDE.md` — directly shapes Claude's behavior in every target repo
- `package.json` — `"files"` controls what ships to npm; `"bin"` wires the CLI name

## Configuration and Secrets

No configuration files. No environment variables. All options are CLI flags. npm publish requires an auth token (not stored in the repo).

## Data and State

Stateless. Pure filesystem I/O against the target directory passed via `--cwd` or `process.cwd()`.

## Testing and Validation

No automated test suite. Manual test plan in README covers: first run, idempotent re-run, `--force`, `--cwd`, and missing directory error. Validate by running `node bin/cli.js --cwd <tmp-dir>`.

## Build, Run, and Deployment

- No build step — source runs directly in Node.js
- Local dev: `npm link` then `create-claude-repo-memory --cwd <dir>`, or `node bin/cli.js --cwd <dir>`
- Release: bump version in `package.json`, `npm publish --access public` (needs npm token with 2FA bypass)

## Known Gotchas

- `getPackageRoot()` uses `import.meta.url` relative to `src/` — moving `paths.js` breaks template resolution
- commander's `--no-gitignore` stores the value as `options.gitignore` (boolean false), not `options.noGitignore` — the destructuring `{ gitignore: updateGitignore }` in `runInit` depends on this
- `BLOCK_MARKER` is `.claude-local/` — if the gitignore block content changes in a future version, existing repos that already have `.claude-local/` in their `.gitignore` won't receive the update
- npm requires a granular access token with "bypass 2FA" enabled for `npm publish`; OTP via `--otp` also works
